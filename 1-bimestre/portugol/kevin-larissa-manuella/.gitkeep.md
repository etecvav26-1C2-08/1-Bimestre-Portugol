## Exercício 1 — Pergunta conceitual

### O que é um algoritmo?
Um algoritmo é uma **sequência finita, organizada e lógica de passos** para realizar uma tarefa ou resolver um problema. É como uma "receita" que guia o processamento de uma informação até um resultado esperado, sem ambiguidades.

### Relação entre lógica de programação e algoritmos
A **lógica de programação** é o raciocínio e a técnica de estruturar pensamentos para atingir um objetivo. O **algoritmo** é a aplicação prática dessa lógica em um plano de ação estruturado.

### Exemplo de algoritmo do cotidiano: Fritar um ovo
1. Pegar uma frigideira.
2. Colocar um pouco de óleo ou manteiga.
3. Acender o fogo.
4. Quebrar o ovo na frigideira.
5. Esperar o ovo fritar (ficar firme).
6. Desligar o fogo.
7. Retirar o ovo e colocar no prato.

## Exercício 2 — Pergunta de aplicação

### Algoritmo em linguagem natural
1. Leia o primeiro número (N1), o segundo (N2) e o terceiro (N3).
2. Defina uma variável chamada `maior` e atribua a ela o valor de N1.
3. Se N2 for maior que `maior`, então `maior` recebe o valor de N2.
4. Se N3 for maior que `maior`, então `maior` recebe o valor de N3.
5. Exiba o valor da variável `maior`.

### Representação em Portugol
```portugol
programa {
  funcao inicio() {
    real n1, n2, n3, maior

    escreva("Digite o primeiro número: ")
    leia(n1)
    escreva("Digite o segundo número: ")
    leia(n2)
    escreva("Digite o terceiro número: ")
    leia(n3)

    maior = n1

    se (n2 > maior) {
      maior = n2
    }
    se (n3 > maior) {
      maior = n3
    }

    escreva("O maior número é: ", maior)
  }
}
```
## Exercício 3 — Pergunta de investigação

### Sobre o Portugol Studio
O **Portugol Studio** é um ambiente de desenvolvimento (IDE) gratuito e brasileiro, desenvolvido pela Univali, focado no ensino de lógica de programação. Ele utiliza a linguagem **Portugol**, que permite escrever comandos em português (como `escreva`, `leia`, `se`, `enquanto`), eliminando a barreira do idioma inglês para quem está começando e permitindo foco total na estruturação do raciocínio lógico.

### Trecho do código criado (Multiplicação por 2)
```portugol
programa {
  funcao inicio() {
    // Declaração de variáveis
    inteiro numero, resultado

    escreva("Digite um número inteiro: ")
    leia(numero)

    // Processamento do cálculo
    resultado = numero * 2

    // Exibição do resultado
    escreva("O dobro de ", numero, " é igual a: ", resultado)
  }
}
```
## Exercício 4 — Pergunta de reflexão

Importância das ferramentas (IDEs, Bibliotecas e Frameworks)
Essas ferramentas são fundamentais porque funcionam como multiplicadores de produtividade. Uma IDE automatiza a organização do código e a detecção de erros; bibliotecas e frameworks oferecem soluções prontas para problemas comuns (como conexão com bancos de dados ou interfaces visuais). Elas permitem que o desenvolvedor concentre seus esforços na lógica de negócio única do seu projeto, em vez de gastar tempo resolvendo problemas técnicos que já possuem soluções padrão.

### Problemas de desenvolver todos os componentes do zero
**Inviabilidade de Tempo:** Projetos que hoje levam semanas demorariam anos se fosse necessário programar desde o comportamento básico do hardware até a interface final.

**Risco de Segurança:** Componentes prontos são testados exaustivamente por milhares de especialistas. Criar um sistema de segurança ou criptografia do zero aumenta drasticamente a chance de falhas críticas.

**Dificuldade de Manutenção:** Sem padrões definidos por frameworks, o código se torna confuso e quase impossível de ser compreendido ou consertado por outros desenvolvedores no futuro.

## Justificativa com exemplos: 

Um programador que utiliza um framework é como um engenheiro civil que compra vigas de aço e cimento certificados: ele foca no design e na estrutura do prédio. Se ele tivesse que fabricar seus próprios pregos e processar a própria matéria-prima do concreto antes de começar a obra, o custo seria impeditivo e a segurança da construção seria incerta.
